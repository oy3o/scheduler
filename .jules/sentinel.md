## 2024-05-24 - Stack Trace Leakage in Public APIs
**Vulnerability:** Unhandled panics in tasks executing inside the Gatekeeper were wrapping the `debug.Stack()` output and directly returning it to callers of `Future.Get(ctx)`, leaking sensitive internal system architecture and paths to potentially untrusted consumers. Additionally, `os.Stderr` was hardcoded to print timeout warnings containing internal state.
**Learning:** Reusable library code needs strict isolation between internal telemetry mechanisms (which need maximum context to debug issues) and public-facing APIs (which need sanitized errors to prevent information leakage). The Gatekeeper had an `OnError` hook configured for logging but failed to use a separated representation for the caller itself.
**Prevention:** When recovering from panics in a public package, split the error representation: provide a sanitized error without stack traces to the public caller, and capture the detailed stack trace exclusively for backend observability systems or configured telemetry hooks. Never hardcode `os.Stderr` logging in reusable libraries; rely exclusively on explicit user-configured hooks like `OnError`/`OnPanic`.
